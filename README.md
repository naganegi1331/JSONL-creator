# LLM学習データ蓄積GUIツール

業務で得たナレッジやトラブルシューティングの記録をGUIから入力し、
LLMファインチューニング用のデータ（Alpaca形式）として蓄積するツールです。
データは **SQLite** で管理し、必要なときに **JSONL** へエクスポートできます。

## 特徴

- Tkinter / sqlite3（いずれもPython標準ライブラリ）製で外部依存なし
- Instruction / Input / Output の3欄を入力して1レコードずつ保存
- 保存済みデータの **一覧表示・編集・削除（複数選択可）** に対応
- ボタン一つでDBの全件を `training_data.jsonl`（UTF-8 / `ensure_ascii=False`）へエクスポート
- **Ollama** のローカル埋め込みモデルでデータをベクトル化し、類似データ検索や
  インポート時の類似重複検出に利用可能（任意機能・Ollama未使用でも動作）
- exe化（PyInstaller等）してもスクリプト実行でも同じディレクトリにDB・出力を作成

## 動作環境

- 対象OS: Windows 10 / 11
- Python 3.x（Tkinter同梱の標準的なインストールでそのまま動作）

## 使い方

```sh
python jsonl_creator.py
```

左ペインに保存済みデータの一覧、右ペインに入力フォームが表示されます。

1. **Instruction**（必須）: 解決すべきメインの課題や質問文
2. **Input**（任意）: 前提条件、環境、エラーコード、ログなどの補足情報
3. **Output**（必須）: 最終的な回答、解決手順、社内仕様などの正解データ
4. **Source**（任意）: 引用元の参考URLや社内ユーザーガイドの参照先など

### ボタン

- **保存**: 新規入力中はDBへ追加、一覧から行を選択して編集中はその行を更新します。
  InstructionまたはOutputが空欄（空白のみを含む）の場合は警告を表示して中断します。
- **新規入力**: フォームをクリアして新規入力モードに戻します。
- **選択行を削除**: 一覧で選択中のレコードを確認のうえ削除します。Ctrl・Shiftキーで
  複数行を選択すれば、まとめて削除できます。
- **JSONLをインポート**: ファイルを選択してDBへ取り込みます（重複の扱いは下記参照）。
- **JSONLにエクスポート**: DB内の全レコードを `training_data.jsonl` に書き出します。
- **ベクトル化（未処理分を一括処理・Ollama使用）**: まだベクトル化していない
  レコードをOllamaの埋め込みモデルでまとめてベクトル化し、DBに保存します
  （詳細は下記「Ollamaによるベクトル化」を参照）。

一覧の行を1行だけクリックすると内容がフォームに読み込まれ、編集できます
（複数選択中はフォームへの読み込みは行わず、削除のみ可能です）。
一覧の上にある検索欄に質問やキーワードを入力して「類似検索」を押すと、
ベクトル化済みのデータの中から類似度の高いものを上位20件表示します。
「一覧に戻る」で通常の一覧表示に戻れます。

## インポートと重複の扱い

「JSONLをインポート」で任意のJSONLファイルをDBへ取り込めます。取り込み時、
**instruction・input・output の3フィールドがすべて一致するレコードは重複として
スキップ**します（DB内の既存データ・ファイル内の先行行の両方が対象）。
同じ質問でも回答が異なるデータは重複とみなさず追加します。`source` は
重複判定に含めないため、同じ三つ組で引用元だけが異なる場合は先に取り込んだ
ものが残ります。`source` キーが無い行は空文字として取り込みます。

必須項目（instruction / output）が空のもの、JSONとして解析できない行は
不正としてスキップします。完了時に「追加 / 重複スキップ / 不正スキップ /
類似重複」の件数を表示します（Ollamaを利用できない場合、類似重複は常に0件
になります）。

## Ollamaによるベクトル化

[Ollama](https://ollama.com/) をローカルで起動し、埋め込みモデル（既定では
`nomic-embed-text`）を事前に取得しておくと、以下の機能が使えます。

```sh
ollama pull nomic-embed-text
```

- **保存**: レコードの内容（instruction / input）をベクトル化してDBに保存します
  （将来のRAG用途などを想定。JSONLエクスポートにはベクトルを含めません）。
  保存・インポート直後は未処理のままで、「ベクトル化（未処理分を一括処理）」
  ボタンで後からまとめてベクトル化する運用です。編集してレコードの内容を
  変更した場合、古いベクトルは破棄され、再度の一括処理が必要になります。
- **インポート時の類似重複検出**: 「JSONLをインポート」時、取り込む行と既存
  データのコサイン類似度が0.93以上の場合に「類似重複」として件数を報告します。
  完全一致でないデータは削除・スキップせず、すべて追加したうえで件数のみ
  知らせます（要確認は利用者の判断に委ねます）。
- **類似データ検索**: 一覧上部の検索欄にキーワードを入力して類似データを
  検索できます。

Ollamaが起動していない場合、これらのベクトル化関連機能は失敗しますが、
保存・編集・削除・JSONLインポート/エクスポートなどの基本機能には影響しません
（インポート時はベクトル化のみスキップされ、通常の取り込みは継続します）。
データ量が多い場合、一括ベクトル化は時間がかかり、処理中はGUIの操作を
受け付けません。

## データ管理

- 保存先DB: 実行ファイル/スクリプトと同じディレクトリの `training_data.db`
- 初回起動時、同ディレクトリに既存の `training_data.jsonl` があれば自動でDBへ取り込みます
  （DB新規作成時のみ。以降は取り込みません。重複スキップは手動インポートと同じ）

## 出力形式

エクスポートすると、1レコードにつき以下のJSONオブジェクトを1行ずつ書き出します。
学習の三つ組（instruction / input / output）はそのまま保ち、引用元は `source`
という別キーのメタ情報として付与します（未入力時は空文字）。学習に使う際に
`source` を無視・除去すれば、純粋なAlpaca形式として扱えます。

```json
{"instruction": "入力値", "input": "入力値(空の場合は空文字)", "output": "入力値", "source": "引用元(空の場合は空文字)"}
```

## exe化

`build.bat` をダブルクリック（またはコマンドプロンプトで実行）すると、
PyInstallerの導入確認・インストールからビルドまでを自動で行います。

```sh
build.bat
```

手動で行う場合は以下のコマンドでも同様にビルドできます。

```sh
pip install pyinstaller
pyinstaller --onefile --noconsole --name jsonl_creator jsonl_creator.py
```

生成された `dist/jsonl_creator.exe` を実行すると、exeと同じディレクトリに
`training_data.db` が作成され、エクスポート時に `training_data.jsonl` が出力されます。
